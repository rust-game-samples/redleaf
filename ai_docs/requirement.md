# RedLeaf CMS - 要件定義

## プロジェクト概要

**RedLeaf** は Rust 製の軽量 CMS（コンテンツ管理システム）。WordPress の代替を目指し、以下の価値を提供する。

| 価値 | 説明 |
|---|---|
| 高速 | Rust コンパイル済みバイナリによる低レイテンシ |
| 軽量 | シングルバイナリ・SQLite で依存インフラ最小化 |
| 拡張性 | プラグイン・テーマシステム（将来対応） |
| API ファースト | REST/GraphQL でヘッドレス CMS として利用可能 |

## 非機能要件

### パフォーマンス

- 単一バイナリで起動（実行環境への依存なし）
- SQLite によるゼロインフラ運用
- Tokio 非同期ランタイムで高並列リクエスト処理
- リリースビルドは LTO + O3 最適化

### セキュリティ

- パスワードは Argon2id でハッシュ（プレーンテキスト保存禁止）
- SQL クエリはすべてプリペアドステートメント
- JWT シークレットは環境変数で管理（ソースに埋め込まない）
- 管理画面は認証ミドルウェアで保護
- XSS 対策：テンプレートの自動エスケープ

### 可用性・運用

- `.env` ファイルで設定を外部化
- `RUST_LOG` でログレベル制御
- `sqlx::migrate!` によるスキーマバージョン管理
- ヘルスチェックエンドポイント（将来）

### 開発体験

- コンパイル時 SQL チェック（SQLx）
- 型安全な ORM（SQLx FromRow）
- エラーは `anyhow::Result` で伝播・`tracing` でログ
- ホットリロード対応（`cargo-watch` 活用を想定）

## 制約・前提

| 項目 | 内容 |
|---|---|
| データベース | SQLite（初期フェーズ）。スケール時に PostgreSQL 移行を検討 |
| 言語 | Rust 2021 edition |
| 最小動作環境 | Linux / macOS（Windows は未確認） |
| ポート | デフォルト 3000（環境変数 `PORT` で変更可） |
| ファイルベース DB | SQLite ファイルはバックアップが容易 |

## ユーザーロール（将来）

| ロール | 権限 |
|---|---|
| 管理者 (admin) | 全操作（投稿・ユーザー・設定） |
| 編集者 (editor) | 全投稿の編集・公開 |
| 著者 (author) | 自分の投稿のみ管理 |
| 購読者 (subscriber) | コンテンツ閲覧のみ |
