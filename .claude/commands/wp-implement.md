WordPress parity タスクを実装します。

実装対象: $ARGUMENTS

## 手順

1. `ai_docs/wordpress-parity-tasks.md` を読んで全タスクを把握する
2. `$ARGUMENTS` が空の場合:
   - フェーズ別に未完了タスク（`- [ ]`）をすべてリストアップして表示する
   - 「実装するタスクを `/wp-implement <タスク名またはフェーズ番号>` で指定してください」と案内する
3. `$ARGUMENTS` が指定されている場合:
   - 対象タスクを特定する（部分一致・フェーズ名でも可）
   - 複数タスクが該当する場合はリストを表示して確認を取る
   - 実装を進める

## 実装方針

- 実装前に関連ファイルを読んで既存パターンを把握する
- 既存コードのスタイルに合わせる（Axum, Askama, SQLx のパターン）
- 新しいマイグレーションが必要な場合は連番ファイル名を使う
- 実装完了後は必ず `cargo build` でコンパイルを確認する
- ビルドが通ったら `ai_docs/wordpress-parity-tasks.md` の対象タスクを `- [x]` に更新する
- 実装サマリーを報告する（変更ファイル一覧 + 追加機能の使い方）

## 注意事項

- 一度に実装するタスクは関連する1グループまでにする（大きすぎる場合は分割を提案する）
- セキュリティに関わる変更（認証・権限チェック）は慎重に実装する
- 既存のテスト (`tests/`) が壊れないように注意する
